SAE 203 - Produire un site web avec une Base de Données

Le Brief

Réalisation d’un site web autour d’un thème imposé : les cafés historiques.

J’ai développé toutes les pages du site conformément au cahier des charges, en y intégrant une base de données MySQL et un espace d’administration permettant de gérer les contenus. J’ai également produit un dossier d’analyse approfondi, allant du choix du style graphique aux tests de performance, en passant par l’étude des données.

Consulter le dossier d’analyse (PDF)
01

Page d'accueil

Pour l’ensemble du site, j’ai choisi une palette marron et crème afin de rappeler l’univers du café. La page d’accueil intègre un hero immersif représentant l’intérieur d’un café historique, permettant à l’utilisateur d’entrer immédiatement dans l’ambiance, avec un bouton menant à la page de recherche. J’ai ensuite intégré une section mettant en avant trois villes historiques, chacune accompagnée d’une photo, d’un court texte et d’un accès à la page des données. Une seconde section est consacrée aux personnalités ayant fréquenté ces cafés, toujours avec un bouton permettant de rejoindre la page dédiée.

02

Page données

La page de données repose sur une base MySQL structurée autour de deux tables : une consacrée aux cafés historiques et une autre aux personnalités. La table des cafés rassemble toutes les informations essentielles : nom, date de création, pays, style architectural, personnalités associées, adresse, coordonnées et photo. La table des personnalités, quant à elle, inclut leur nom, prénom, profession, le café qu’elles fréquentaient et la date de leur première visite.

03

Page recherche

La page de recherche permet aux utilisateurs d’explorer les cafés en fonction des pays. J’y ai intégré une carte interactive connectée à la base MySQL, qui se met automatiquement à jour lors de l’ajout d’un nouveau café. Chaque point de la carte affiche le nom du café, son pays ainsi qu’une photo, pour mieux visualiser les données.
